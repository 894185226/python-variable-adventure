# Python 变量专题学习网站

## 一、项目简介

本项目是一个面向中学生的 Python 变量专题互动学习网站，采用 **B/S 架构**，集成了以下技术：

| 技术层次 | 技术选型 |
|----------|----------|
| 前端 | HTML5 + CSS3 + JavaScript（W3.CSS 风格） |
| 后端 | Node.js + Express |
| 数据库 | MySQL 5.7 / 8.0 / 8.4 / 9.x |
| 版本管理 | Git + Gitee + GitHub |

网站包含 **10 个学习模块**，涵盖情境导入、知识讲解、实践操作、错误调试、课堂小测等教学环节，并内置 **8 项成就系统**，通过闯关激励机制提升学生学习兴趣。获得成就时，页面右下角会弹出 Steam 风格的成就提示框。

**UI/UX 特色**：首页采用 W3Schools 风格彩色分栏布局，支持亮色/暗色主题一键切换，内置打字机代码演示效果、滚动渐入动画、卡片悬浮微交互等，兼顾简约感与精致质感。

## 二、快速开始（一键启动）

项目已配置 **全自动启动脚本**，只需一个步骤：

> **双击 `start_server.bat`**

脚本会自动完成以下工作（5 个阶段）：

| 阶段 | 说明 |
|------|------|
| Phase 1：Node.js | 自动查找已有安装 → 搜索常见目录 → winget 自动安装（含镜像回退） → 刷新 PATH |
| Phase 2：npm 依赖 | 自动执行 `npm install`，失败时自动切换国内镜像 |
| Phase 3：MySQL 检测 | 搜索全版本安装路径（5.7 ~ 9.7），未找到则 winget 自动安装 |
| Phase 4：MySQL 配置 | 自动初始化数据目录（`--initialize-insecure`） → 创建 Windows 服务（`--install`） → 启动服务并轮询等待 → 连接失败则通过安全模式重置 root 密码为空 |
| Phase 5：启动网站 | 启动 Node.js 后端服务器，`server.js` 自动创建数据库和全部数据表 |

> **[首次启动]** 脚本会自动请求管理员权限（UAC 弹窗），请点击"是"。全新电脑首次启动可能需要几分钟（自动下载安装 Node.js、MySQL、npm 依赖）。

启动成功后，命令行窗口显示：

```
===========================================
  Python 变量学习平台已启动
  地址：http://localhost:3000
  数据库：MySQL -> python_var_lesson
  关闭：Ctrl + C
===========================================
```

在浏览器中访问 **http://localhost:3000** 即可使用。

## 三、环境要求

### 3.1 最低配置

| 软件 | 说明 |
|------|------|
| Windows 10 / 11 | 操作系统（需支持 winget 包管理器） |
| Node.js 16+ | 双击 bat 脚本可自动安装 |
| MySQL 5.7 ~ 9.7 | 双击 bat 脚本可自动安装并完成全部配置 |

### 3.2 MySQL 重要配置

`server.js` 中 MySQL 连接配置默认为：

```javascript
{
    host: 'localhost',
    user: 'root',
    password: '',   // 空密码
    database: 'python_var_lesson'
}
```

> **如果安装 MySQL 时设置了 root 密码**，脚本会自动尝试通过安全模式将其重置为空密码。如果自动重置失败，请用记事本打开 `server.js`，将 `password: ''` 改为你设置的密码。

## 四、跨电脑移植

### 4.1 复制文件

将整个项目文件夹复制到目标电脑上的任意位置。**不要只复制单个文件**，需要保证以下文件齐全：

| 文件 | 说明 |
|------|------|
| `start_server.bat` | 一键启动脚本（核心） |
| `server.js` | Node.js 后端服务器（含自动建库建表） |
| `index.html` | 网站主页 |
| `style.css` | 样式文件 |
| `script.js` | 前端交互逻辑 |
| `package.json` | Node.js 依赖配置 |
| `database.sql` | 数据库建表脚本（备用） |

### 4.2 在新电脑上启动

双击 `start_server.bat`，等待自动配置完成即可。脚本会自动处理：

- 检测或安装 Node.js
- 检测或安装 MySQL
- 初始化 MySQL 数据目录
- 创建并启动 MySQL Windows 服务
- 重置 MySQL root 密码为空
- 安装 npm 依赖
- 创建数据库和表
- 启动网站服务器

### 4.3 注意事项

- 确保目标电脑可以访问互联网（首次启动需要下载依赖）
- 首次启动时可能弹出 UAC 窗口，请点击"是"以授予管理员权限
- 如果通过 U 盘或网盘传输文件，直接复制文件夹即可，不需要做额外处理
- 如果两台电脑使用不同的 MySQL root 密码，脚本会自动尝试重置，无需手动干预

## 九、常见问题

### Q1：双击 start_server.bat 后窗口闪退？

可能原因及解决：

1. **Node.js 未安装且 winget 安装失败** —— 手动安装 Node.js：https://nodejs.org/
2. **MySQL 未安装且 winget 安装失败** —— 手动安装 MySQL（安装时设 root 密码为空）：https://dev.mysql.com/downloads/installer/
3. **文件换行符错误**（例如通过某些工具编辑过 bat 文件）—— Windows 批处理文件必须使用 CRLF 换行符

### Q2：打开网站后登录显示"服务器错误"？

1. 确保命令行窗口没有关闭（服务器正在运行）
2. 检查窗口中是否有 `[错误] 数据库初始化失败` 的提示
3. 确认 MySQL 服务已启动 —— 打开 Windows 服务管理器（services.msc），查找 MySQL 服务并确保其状态为"正在运行"
4. 确认 MySQL root 密码为空，或已修改 `server.js` 中的 `password` 配置

### Q3：启动时提示需要管理员权限？

脚本已内置自动提权功能。双击 bat 后如果弹出 UAC（用户账户控制）窗口，点击"是"即可。如果 UAC 窗口被拒绝，可以右键点击 `start_server.bat` → "以管理员身份运行"。

### Q4：MySQL 安装后一直连不上？

脚本在 Phase 4 中会自动尝试以下操作：

1. 通过 Windows 服务启动 MySQL
2. 直接启动 mysqld 进程
3. 如均失败，通过安全模式（`--skip-grant-tables`）重置 root 密码为空

如果以上步骤全部失败，请手动检查：

- 打开 services.msc，确认 MySQL 服务存在且可启动
- 确认 MySQL 数据目录已初始化（`C:\ProgramData\MySQL\` 下应有 Data 文件夹）
- 尝试手动执行：`mysqld --initialize-insecure --console`

### Q5：数据库中文乱码？

`server.js` 已配置 `charset: 'utf8mb4'`，数据库自动创建时也指定了 `utf8mb4_unicode_ci` 编码，一般不会出现乱码。如仍有问题，检查 MySQL 默认编码设置。

### Q6：如何查看注册账号的数据？

可使用 Navicat、DBeaver 等工具连接本地 MySQL（地址 localhost，用户 root，空密码），查看 `python_var_lesson` 数据库中的 `students`、`learning_progress`、`achievements`、`login_logs` 表。

### Q7：如何重置某位学生的数据？

在 MySQL 命令行中执行（以用户名为 `test001` 为例）：

```sql
SET @sid = (SELECT id FROM python_var_lesson.students WHERE username = 'test001');
DELETE FROM python_var_lesson.learning_progress WHERE student_id = @sid;
DELETE FROM python_var_lesson.achievements WHERE student_id = @sid;
DELETE FROM python_var_lesson.login_logs WHERE student_id = @sid;
DELETE FROM python_var_lesson.students WHERE id = @sid;
```
